# DonNext Microservices

Three FastAPI services + a real-time KPI simulator.

```
microservices/
├── explainability/
│   ├── app.py              ← SHAP + feature importance  (port 8001)
│   ├── requirements.txt
│   └── Dockerfile
├── monitoring/
│   ├── app.py              ← cluster health & alerts    (port 8002)
│   ├── requirements.txt
│   └── Dockerfile
├── prediction/
│   ├── app.py              ← 4-target HO predictions    (port 8003)
│   ├── requirements.txt
│   └── Dockerfile
├── simulator/
│   ├── simulator.py        ← pushes real-time KPI data to all 3
│   └── requirements.txt
└── docker-compose.yml
```

---

## Option A — Run with Docker (recommended)

```bash
# 1. Start all three services
cd microservices
docker compose up --build

# 2. In a separate terminal, run the simulator
cd microservices/simulator
pip install -r requirements.txt
python simulator.py
```

---

## Option B — Run locally without Docker

Open four terminals:

**Terminal 1 — Explainability (port 8001)**
```bash
cd microservices/explainability
pip install -r requirements.txt
uvicorn app:app --reload --port 8001
```

**Terminal 2 — Monitoring (port 8002)**
```bash
cd microservices/monitoring
pip install -r requirements.txt
uvicorn app:app --reload --port 8002
```

**Terminal 3 — Prediction (port 8003)**
```bash
cd microservices/prediction
pip install -r requirements.txt
uvicorn app:app --reload --port 8003
```

**Terminal 4 — Simulator**
```bash
cd microservices/simulator
pip install -r requirements.txt
python simulator.py
```

---

## Simulator options

```bash
python simulator.py                        # default: all clusters, 3s interval
python simulator.py --interval 1           # push every second
python simulator.py --clusters 77 78 79    # only specific clusters
python simulator.py --once                 # one push then exit (for testing)
```

## Key endpoints

### Explainability :8001
- POST `/api/v1/explainability/cluster/ingest?cluster_id=77`  ← simulator
- GET  `/api/v1/explainability/cluster/{id}`
- GET  `/api/v1/explainability/cluster/{id}/shap`
- GET  `/api/v1/explainability/cluster/{id}/feature-importance`
- GET  `/api/v1/explainability/summary`

### Monitoring :8002
- POST `/api/v1/metrics/ingest`  ← simulator
- GET  `/api/v1/system/health`
- GET  `/api/v1/clusters/{id}/metrics`
- GET  `/api/v1/alerts`
- GET  `/api/v1/metrics/trends`

### Prediction :8003
- POST `/api/v1/clusters/ingest`  ← simulator
- GET  `/api/v1/clusters/{id}/predict`
- GET  `/api/v1/predictions/summary`
- GET  `/api/v1/models/compare`
- POST `/api/v1/models/{name}/retrain`

## Data flow

```
simulator.py
    ├── POST /api/v1/metrics/ingest                        → monitoring :8002
    ├── POST /api/v1/clusters/ingest                       → prediction :8003
    └── POST /api/v1/explainability/cluster/ingest         → explainability :8001
```

If the simulator is not running the services fall back to random data,
so the frontend always has something to display.
