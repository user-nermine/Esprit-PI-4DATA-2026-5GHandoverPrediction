HOW TO USE THIS FILE
────────────────────
1. Open Postman
2. For each test: create a new request, copy the method + URL + headers + body
3. Run them IN ORDER — some tests depend on results of previous ones
4. After LOGIN, copy the token and paste it in all "Authorization" headers

TEST 1 — REGISTER (create a new user account)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHAT IT DOES:
  Creates a brand new user in the database.
  The password is hashed with BCrypt before saving.
  A JWT token is returned immediately — you are "logged in" right after registering.
  No token needed to call this endpoint (it's public).

METHOD:  POST
URL:     http://localhost:8081/api/auth/register

HEADERS:
  Content-Type: application/json

BODY (raw JSON):
{
  "fullName": "Nermine Rahalli",
  "email": "nermine@esprit.tn",
  "password": "Test1234!",
  "role": "RAN_ENGINEER"
}

VALID ROLES (copy exactly):
  SYSTEM_ADMIN
  DATA_SCIENTIST
  RAN_ENGINEER
  NOC_ENGINEER

EXPECTED RESPONSE — 201 Created:
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "email": "nermine@esprit.tn",
  "fullName": "Nermine Rahalli",
  "role": "RAN_ENGINEER"
}

POSSIBLE ERRORS:
  400 Bad Request  → "Email is invalid" or "Password must be at least 8 characters"
                     → Check your JSON body matches the format exactly
  500 Internal     → "Email already exists: nermine@esprit.tn"
                     → That email is already registered, change it

TEST 3 — LOGIN (get your JWT token)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHAT IT DOES:
  Authenticates an existing user with email + password.
  The backend looks up the user, checks password with BCrypt.matches(),
  updates the lastLogin timestamp, then returns a fresh JWT token.
  This token is valid for 24 hours (86400000 ms in JwtService).

  *** IMPORTANT: Copy the "token" value from the response.
  *** You will paste it in ALL subsequent tests as the Authorization header.

METHOD:  POST
URL:     http://localhost:8081/api/auth/login

HEADERS:
  Content-Type: application/json

BODY (raw JSON):
{
  "email": "nermine@esprit.tn",
  "password": "Test1234!"
}

EXPECTED RESPONSE — 200 OK:
{
  "token": "eyJhbGciOiJIUzI1NiJ9.eyJyb2xlIjoiUkFOX0VOR0lORUVSIiwic3ViIjoibmVybWluZUBlc3ByaXQudG4iLCJpYXQiOjE3NDUyNDI0MDAsImV4cCI6MTc0NTMyODgwMH0.XXXX",
  "email": "nermine@esprit.tn",
  "fullName": "Nermine Rahalli",
  "role": "RAN_ENGINEER"
}

HOW TO USE THE TOKEN IN POSTMAN:
  In every next request, go to the "Headers" tab and add:
  Key:   Authorization
  Value: Bearer eyJhbGciOiJIUzI1NiJ9...   (paste your full token after "Bearer ")

  OR use the "Authorization" tab in Postman:
  Type: Bearer Token → paste token in the field

POSSIBLE ERRORS:
  500 → "Invalid email or password" — wrong email or password
  500 → "Invalid email or password" — user does not exist (register first)
